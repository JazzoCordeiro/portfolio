do y while en JavaScript

Los bucles do y while en JavaScript se usan para ejecutar repetidamente un bloque de código mientras se cumpla una condición especificada como true. Sin embargo, hay una diferencia clave entre ellos: la comprobación de la condición.

Bucle while

El bucle while en JavaScript verifica la condición antes de cada iteración. Si la condición es true, el bucle se ejecutará. Si la condición es false desde el principio, el bucle nunca se ejecutará.

Sintaxis del bucle while:

while (condición) {
      // Código a ejecutar mientras la condición sea verdadera
    }

Ejemplo de un bucle while:

let count = 0;
    
    while (count < 5) {
      console.log(count);  // Esto imprimirá los números del 0 al 4
      count++;  // Incrementa el contador
    }

En este ejemplo:

  • El bucle se ejecutará mientras count sea menor que 5.
  • El count++ dentro del bucle incrementa el valor de count en cada iteración.
  • Cuando count alcance 5, la condición count < 5 se volverá false, y el bucle se detendrá.

Bucle do...while

El bucle do...while, a diferencia del bucle while, garantiza que el bloque de código dentro del bucle se ejecute al menos una vez, sin importar la condición. Esto se debe a que la condición se comprueba después de que se ejecute el bloque de código.

Sintaxis del bucle do...while:

do {
      // Código a ejecutar
    } while (condición);

Ejemplo de un bucle do...while:

let count = 0;
    
    do {
      console.log(count);  // Esto imprimirá los números del 0 al 4
      count++;  // Incrementa el contador
    } while (count < 5);

En este ejemplo:

  • El bloque do se ejecutará primero, imprimiendo 0 en la primera iteración.
  • Luego, después de ejecutar el bloque, verifica si la condición count < 5 es true. Si es así, el bucle continúa. Si la condición es false, el bucle se detiene.
  • Aunque la condición fuera inicialmente false, el bucle aún se ejecutaría una vez antes de comprobarla.

Comparación: do...while vs while

Característica Bucle while Bucle do...while
Comprobación de la condición Comprueba la condición antes de ejecutar el bloque de código Comprueba la condición después de ejecutar el bloque de código
Ejecutación garantizada El código puede no ejecutarse si la condición es falsa desde el inicio El código siempre se ejecuta al menos una vez
Uso recomendado Útil cuando necesitas comprobar una condición antes de ejecutar el bloque de código Útil cuando necesitas asegurarte de que el bloque de código se ejecute al menos una vez, incluso si la condición es falsa

Ejemplo donde do...while es útil:

Considera el caso en el que queremos pedirle al usuario un número hasta que ingrese uno válido mayor que 0:

let number;
    
    do {
      number = prompt("Introduce un número mayor que 0:");
    } while (number <= 0);
    
    console.log("Introdujiste un número válido:", number);

En este ejemplo, el código dentro del bloque do se ejecutará al menos una vez, pidiendo al usuario que ingrese un número. Si el usuario introduce un número mayor que 0, el bucle se detendrá; de lo contrario, se pedirá al usuario nuevamente.

Resumen:

  • Bucle while: La condición se comprueba antes de ejecutar el código. Si la condición es falsa al principio, el código dentro del bucle no se ejecutará.
  • Bucle do...while: El código se ejecuta al menos una vez antes de comprobar la condición. Incluso si la condición es falsa al principio, el código dentro del bucle se ejecutará una vez.

Estos bucles son útiles para diferentes escenarios dependiendo de si deseas que el bucle se ejecute al menos una vez o comprobar la condición primero.
