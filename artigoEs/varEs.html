¿Qué Son las Variables?

Las variables son espacios de memoria con nombre que almacenan datos. Permiten almacenar y manipular valores en tu código, y pueden ser utilizadas en diferentes partes del programa. En JavaScript, una variable puede declararse usando las palabras clave var, let o const.

var: Esta era la forma tradicional de declarar variables en JavaScript. Sin embargo, puede causar problemas con el alcance (scope) porque tiene un alcance de función y permite reasignaciones. Su uso ahora está desaconsejado en favor de mejores alternativas.

let: Declara una variable con alcance de bloque, lo que significa que la variable existe solo dentro del bloque en el que fue declarada. Puede ser reasignada.

const: Declara una variable cuyo valor no puede cambiar después de su asignación inicial. También tiene un alcance de bloque.

¿Cuándo Usar Cada Una?

Usa var solo si es necesario para la compatibilidad con versiones antiguas de JavaScript o contextos específicos.

Usa let para declarar variables cuyo valor puede cambiar, y cuando necesites un alcance de bloque.

Usa const siempre que declares una variable cuyo valor no deba alterarse después de la asignación inicial, especialmente para hacer que el código sea más predecible y fácil de entender.

Al adoptar const por defecto y usar let cuando sea necesario, crearás un código más seguro, menos propenso a errores y más fácil de entender.

Declaración de una Variable

La declaración de una variable comienza con una palabra clave que especifica el tipo de variable (como var, let o const) seguida del nombre de la variable. Dependiendo de la palabra clave utilizada, se define el alcance y el comportamiento de reasignación de la variable.

Ejemplos Prácticos

          
            var nombre = "Ana"; // Cadena
            const edad = 25; // Número
            var estaActivo = true; // Booleano
            let persona = { nombre: "Juan", edad: 30 }; // Objeto
            const numeros = [1, 2, 3]; // Arreglo
            let simbolo = Symbol("id"); // Símbolo
            const numeroGrande = 12345678901234567890n; // BigInt

Inicialización y Reasignación

La inicialización es el proceso de asignar un valor a una variable en el momento de su declaración. La reasignación es el cambio del valor de una variable ya existente. Las variables declaradas con let pueden ser reasignadas, pero las variables declaradas con const no pueden serlo.

          
            let x = 10;     // Inicialización
            x = 20;         // Reasignación

ADVERTENCIA: const no puede tener su valor reasignado; es inmutable.

Reglas para Nombrar una Variable

  • Comienza con una letra, guion bajo (_) o signo de dólar ($):
  • Pueden contener letras, números, guiones bajos (_) y signos de dólar ($):
  • Es sensible a mayúsculas y minúsculas:
  • Longitud del nombre: No hay un límite de tamaño, pero debe ser significativo y conciso.
  • Nombres descriptivos: Se prefieren nombres que describan lo que la variable representa.
  • Convenciones de nomenclatura: Aunque no son requeridas por JavaScript, existen algunas convenciones ampliamente seguidas para hacer el código más legible:
  • camelCase: Usado para variables y funciones, donde la primera palabra comienza con letra minúscula y las siguientes palabras con mayúscula.
  • UPPER_SNAKE_CASE: Usado para constantes (en mayúsculas y con guiones bajos entre palabras).

Funciones de JavaScript

A continuación se presentan algunas funciones importantes de JavaScript:

  • eval(): Evalúa código JavaScript representado como una cadena.
  • uneval(): Crea una representación en cadena del código fuente de un objeto.
  • isFinite(): Determina si el valor pasado es un número finito. Si es necesario, el parámetro se convierte primero en un número.
  • isNaN(): Determina si un valor es NaN (No es un número).
  • parseFloat(): Analiza un argumento de cadena y devuelve un número de punto flotante.
  • parseInt(): Analiza un argumento de cadena y devuelve un número entero de la base especificada.
  • decodeURI(): Decodifica un Identificador de Recurso Uniforme (URI) creado previamente por encodeURI.
  • decodeURIComponent(): Decodifica un componente URI creado previamente por encodeURIComponent.
  • encodeURI(): Codifica un URI, reemplazando ciertos caracteres con secuencias de escape que representan la codificación UTF-8 del carácter.
  • encodeURIComponent(): Codifica un componente URI, reemplazando ciertos caracteres con secuencias de escape.
  • escape(): Este método está obsoleto. Reemplaza los caracteres con secuencias de escape hexadecimales. Usa encodeURI o encodeURIComponent en su lugar.
  • unescape(): Este método está obsoleto. Reemplaza las secuencias de escape con el carácter que representan. Usa decodeURI o decodeURIComponent en su lugar.
