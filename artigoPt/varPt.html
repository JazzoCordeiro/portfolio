O que são Variáveis?

Variáveis são espaços nomeados na memória que armazenam dados. Elas permitem armazenar e manipular valores no seu código, e podem ser usadas em diferentes partes do programa. Em JavaScript, uma variável pode ser declarada usando as palavras-chave var, let ou const.

var: Esta era a maneira tradicional de declarar variáveis em JavaScript. No entanto, pode causar problemas com o escopo, pois tem um escopo de função e permite reatribuições. Seu uso agora é desencorajado em favor de alternativas melhores.

let: Declara uma variável com escopo de bloco, ou seja, a variável existe apenas dentro do bloco em que foi declarada. Ela pode ser reatribuída.

const: Declara uma variável cujo valor não pode ser alterado após sua atribuição inicial. Ela também tem escopo de bloco.

Quando Usar Cada Um?

Use var apenas se for necessário para compatibilidade com versões mais antigas do JavaScript ou contextos específicos.

Use let para declarar variáveis cujo valor pode mudar, e quando você precisar de escopo de bloco.

Use const sempre que declarar uma variável cujo valor não deve ser alterado após a atribuição inicial, especialmente para tornar o código mais previsível e fácil de entender.

Adotando const por padrão e usando let quando necessário, você cria um código mais seguro, com menos chances de erro e mais fácil de entender.

Declarando uma Variável

A declaração de uma variável começa com uma palavra-chave que especifica o tipo da variável (como var, let, ou const) seguida pelo nome da variável. Dependendo da palavra-chave usada, o escopo e o comportamento de reatribuição da variável são definidos.

Exemplos Práticos

          
            var nome = "Ana"; // String
            const idade = 25; // Número
            var estaAtivo = true; // Booleano
            let pessoa = { nome: "João", idade: 30 }; // Objeto
            const numeros = [1, 2, 3]; // Array
            let simbolo = Symbol("id"); // Símbolo
            const numeroGrande = 12345678901234567890n; // BigInt

Inicialização e Reatribuição

Inicialização é o processo de atribuir um valor a uma variável no momento de sua declaração. Reatribuição é a mudança do valor de uma variável já existente. Variáveis declaradas com let podem ser reatribuídas, mas variáveis declaradas com const não podem.

          
            let x = 10;     // Inicialização
            x = 20;         // Reatribuição

AVISO: const não pode ter seu valor reatribuído; ela é imutável.

Regras para Nomear uma Variável

  • Comece com uma letra, sublinhado (_) ou cifrão ($):
  • Pode conter letras, números, sublinhados (_) e cifrões ($):
  • Case-sensitive (diferencia maiúsculas de minúsculas):
  • Comprimento do nome: Não há limite de tamanho, mas deve ser significativo e conciso.
  • Nomes descritivos: Prefira nomes que descrevam o que a variável representa.
  • Convenções de nomenclatura: Embora não seja obrigatório no JavaScript, existem algumas convenções amplamente seguidas para tornar o código mais fácil de ler:
  • camelCase: Usado para variáveis e funções, onde a primeira palavra começa com letra minúscula e as palavras subsequentes começam com letra maiúscula.
  • UPPER_SNAKE_CASE: Usado para constantes (em maiúsculas e com underscores entre palavras).

Funções do JavaScript

Aqui estão algumas funções importantes do JavaScript:

  • eval(): Avalia código JavaScript representado como uma string.
  • uneval(): Cria uma representação em string do código fonte de um Objeto.
  • isFinite(): Determina se o valor passado é um número finito. Se necessário, o parâmetro é primeiro convertido para número.
  • isNaN(): Determina se um valor é NaN (Não é um Número).
  • parseFloat(): Analisa um argumento de string e retorna um número de ponto flutuante.
  • parseInt(): Analisa um argumento de string e retorna um número inteiro da base especificada.
  • decodeURI(): Decodifica um Identificador Uniforme de Recurso (URI) previamente criado por encodeURI.
  • decodeURIComponent(): Decodifica um componente de URI previamente criado por encodeURIComponent.
  • encodeURI(): Codifica um URI, substituindo certos caracteres por sequências de escape representando a codificação UTF-8 do caractere.
  • encodeURIComponent(): Codifica um componente de URI, substituindo certos caracteres por sequências de escape.
  • escape(): Este método está obsoleto. Ele substitui caracteres por sequências de escape hexadecimal. Use encodeURI ou encodeURIComponent em vez disso.
  • unescape(): Este método está obsoleto. Ele substitui sequências de escape pelo caractere que representam. Use decodeURI ou decodeURIComponent em vez disso.
