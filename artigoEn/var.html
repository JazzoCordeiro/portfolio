What Are Variables?

Variables are named memory spaces that store data. They allow you to store and manipulate values in your code, and they can be used in different parts of the program. In JavaScript, a variable can be declared using the keywords var, let, or const.

var: This was the traditional way to declare variables in JavaScript. However, it can cause issues with scope because it has a function scope and allows reassignments. Its use is now discouraged in favor of better alternatives.

let: Declares a block-scoped variable, meaning the variable exists only within the block it was declared in. It can be reassigned.

const: Declares a variable whose value cannot be changed after its initial assignment. It also has block scope.

When to Use Each?

Use var only if compatibility with older versions of JavaScript or specific contexts is necessary.

Use let to declare variables whose value can change, and when you need block scope.

Use const whenever declaring a variable whose value should not be altered after the initial assignment, especially to make the code more predictable and easier to understand.

By adopting const by default and using let when necessary, you create safer, less error-prone, and easier-to-understand code.

Declaring a Variable

Variable declaration begins with a keyword that specifies the type of variable (such as var, let, or const) followed by the variable name. Depending on the keyword used, the scope and reassignment behavior of the variable are defined.

Practical Examples

          
            var name = "Ana"; // String
            const age = 25; // Number
            var isActive = true; // Boolean
            let person = { name: "João", age: 30 }; // Object
            const numbers = [1, 2, 3]; // Array
            let symbol = Symbol("id"); // Symbol
            const largeNumber = 12345678901234567890n; // BigInt

Initialization and Reassignment

Initialization is the process of assigning a value to a variable at the time of its declaration. Reassignment is the change of an already existing variable's value. Variables declared with let can be reassigned, but variables declared with const cannot be.

          
            let x = 10;     // Initialization
            x = 20;         // Reassignment

WARNING: const cannot have its value reassigned; it is immutable.

  • Name length: There is no size limit, but it should be meaningful and concise.

  • Naming conventions: Although not required by JavaScript, there are some widely-followed conventions to make the code easier to read:
  • camelCase: Used for variables and functions, where the first word starts with a lowercase letter and subsequent words start with uppercase letters.
  • UPPER_SNAKE_CASE: Used for constants (in uppercase and with underscores between words).

JavaScript Functions

Here are some important JavaScript functions:

  • eval(): Evaluates JavaScript code represented as a string.
  • uneval(): Creates a string representation of the source code of an Object.
  • isFinite(): Determines if the passed value is a finite number. If necessary, the parameter is first converted to a number.
  • isNaN(): Determines if a value is NaN (Not a Number).
  • parseFloat(): Parses a string argument and returns a floating-point number.
  • parseInt(): Parses a string argument and returns an integer of the specified base.
  • decodeURI(): Decodes a Uniform Resource Identifier (URI) previously created by encodeURI.
  • decodeURIComponent(): Decodes a URI component previously created by encodeURIComponent.
  • encodeURI(): Encodes a URI, replacing certain characters with escape sequences representing the UTF-8 encoding of the character.
  • encodeURIComponent(): Encodes a URI component, replacing certain characters with escape sequences.
  • escape(): This method is deprecated. It replaces characters with hexadecimal escape sequences. Use encodeURI or encodeURIComponent instead.
  • unescape(): This method is deprecated. It replaces escape sequences with the character they represent. Use decodeURI or decodeURIComponent instead.
